Dashboard elements

A dashboard is a set of charts that share context. The chrome around them should be calm and predictable so the data stands out.

Anatomy of an NCDPI dashboard

Titles and text

  • Dashboard title: DPI Blue, 26 pt. Subtitle: dark grey, 16 pt.
  • Help text (how to read a chart, how to use a filter): Plum #842175, 11 pt — the same treatment as chart annotations. Help text and annotations are both meta-overlays on data, so they share a color so users learn the convention once.
  • Body text is black or DPI Blue — never color-coded for emphasis.

Common dashboard chrome

Every NCDPI dashboard should carry, somewhere visible (usually the footer):

  • Data last updated — a date, plus the refresh cadence if known (“Updated 2025-09-15, monthly”).
  • Contact — a name or shared inbox for questions and corrections.
  • NCDPI logo — small, top-left or footer, so attribution is unambiguous.
  • Glossary or help link — for any term a non-expert reader might stumble on.

These four items add almost no visual weight but answer the questions every new viewer has within thirty seconds.

Controls and filters

  • Group filters together in a consistent place (top or right rail) so users learn where to look once.
  • Label the selected state clearly (darker) versus unselected (grey).
  • Default to the most useful view, not an empty one — don’t make users configure the dashboard before it says anything.

Keep the filter count low

Every filter is one more decision the user has to make before the dashboard answers anything (Hick’s Law — choice time grows with the number of options). Aim for the minimum set of filters that lets users answer the question the dashboard is built for. If you can’t answer “what does this filter unlock?” in one sentence, the filter probably shouldn’t be there.

When you can’t trim, use progressive disclosure

Sometimes a stakeholder mandates more filters than really fit. In that case, don’t put them all on the page at once — hide secondary filters behind a collapsible panel (Tableau: dynamic zone visibility) or a “More filters” toggle. Show the two or three most-used filters by default and let the user reveal the rest if they need them.

Prefer cross-filtering over filter controls

The most intuitive filter is the chart itself. Let the user click a bar on a top-left chart and watch the rest of the dashboard re-scope to that selection. Cross-filtering replaces chrome with direct manipulation — fewer dropdowns, more discovery.

Layout

Reading order: Z-pattern, top-left first

Western readers scan dashboards in a rough Z: top-left first, eye sweeps across to the top-right, drops down and across again to the bottom-right. The top-left position earns the most attention — put your most important number or chart there.

Drill down as you move down and right

Pair the Z-pattern with a progressive zoom. Top-left should be the 30,000-ft view — the headline number or trend. As the eye moves down and to the right, content gets progressively more granular: state → region → district → school. This matches Shneiderman’s mantra for information visualization: overview first, zoom and filter, details on demand.

Big Aggregate Numbers (BANs) at the top

The headline number for the whole dashboard usually deserves to be the very first thing the eye lands on. A row of BANs — single, large numerals with a short label — anchors the dashboard, sets the scale, and gives the rest of the charts a reference point. Use them sparingly (three to five at most) and label the time period clearly.

BAN page coming soon. BANs are not yet listed as a chart type on this site — a dedicated page is on the roadmap. In the meantime, keep them simple: one big number, one short label, one tiny period of comparison (“↑ 2.1 pts vs. 2022”).

Interactivity earns its keep

Hover for detail, click to highlight, filter to focus. In a dashboard you can show more than in a static chart because the user controls what to look at — but every interactive feature is one more thing to discover. Make the common path obvious.

Dashboards are where highlight-one-on-grey shines: plot every series in grey and let the user click to spotlight the one they care about — no color overload, any number of categories.
